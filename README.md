# ReportAr

منصة عربية لتحويل مسودات التقارير إلى تقارير PDF منسقة باحتراف عبر LaTeX، مع
دعم بوت Telegram، النماذج المحلية عبر Ollama، ونموذج Gemini عبر API.

يركز المشروع على التقارير العربية الرسمية والتنفيذية: ترتيب المحتوى، كشف النقص
والتعارضات، إنشاء مخططات ديناميكية، تطبيق هوية بصرية، ثم إخراج PDF جاهز
للاستخدام.

## القدرات الحالية

- استقبال النصوص وملفات `TXT` و`PDF` و`DOCX` عبر Telegram.
- مساران للعمل:
  - المسار السريع: يختار النمط المناسب وينتج PDF مباشرة.
  - المسار الموجه: يسأل عن النمط والشعار والخلفية والمخططات قبل الاعتماد.
- دعم النماذج:
  - `local`: نموذج محلي عبر Ollama مثل Gemma أو Qwen.
  - `gemini`: نموذج Gemini عبر API.
- لوحة تحكم محلية لحفظ مفاتيح Telegram وGemini وتشغيل البوت.
- قوالب ألوان جاهزة وهوية بصرية محفوظة لكل مستخدم.
- مخططات ديناميكية ثلاثية الأبعاد أو ذات عمق بصري.
- معالجة التقارير الطويلة على مراحل مع الحفاظ على الهوية والأرقام.
- بوابة جودة تمنع ظهور عبارات مثل "تقرير آلي" أو تفاصيل التدقيق داخل التقرير.
- مجموعة اختبارات جودة محلية من 10 سيناريوهات متنوعة.

## التشغيل المحلي

```bash
cd /path/to/report-worker
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

عدّل ملف `.env` وضع مفاتيحك المحلية فقط. لا ترفع هذا الملف إلى GitHub.

لتشغيل لوحة التحكم:

```bash
python -m uvicorn report_worker.dashboard:app --host 127.0.0.1 --port 8765
```

ثم افتح:

```text
http://127.0.0.1:8765
```

يمكن أيضًا تشغيل `start-dashboard.command` على macOS.

## إعداد Telegram وGemini

من لوحة التحكم:

1. ضع مفتاح بوت Telegram.
2. اختر النموذج المحلي أو Gemini.
3. ضع مفتاح Gemini عند الحاجة.
4. اختبر الاتصال.
5. شغّل البوت.

تحفظ الأسرار في `data/settings.json` بصلاحية ملف محدودة، ولا تعيد لوحة التحكم
إظهار قيم المفاتيح بعد حفظها.

## النماذج المحلية

يعتمد الاتصال المحلي على Ollama:

```bash
ollama list
```

يمكن اختيار أي نموذج متاح مثل:

```text
gemma4:31b
qwen3:32b
```

اسم النموذج يكتب في لوحة التحكم أو في `.env`:

```bash
OLLAMA_MODEL=gemma4:31b
```

## البحث عبر الإنترنت

يمكن تفعيل البحث من لوحة التحكم عند الحاجة. عند التفعيل يستخدم النظام Gemini
لجلب مادة موثقة وروابط مصادر، ثم يمرر الخلاصة للنموذج المختار. النموذج المحلي
لا يحصل على اتصال إنترنت مفتوح.

## ملفات التعليمات

تعليمات النموذج ومخطط الاستجابة موجودان داخل:

```text
prompts/report-system-prompt.txt
prompts/report-response.schema.json
```

يمكن تغيير المسارات من `.env`:

```bash
REPORT_SYSTEM_PROMPT=./prompts/report-system-prompt.txt
REPORT_RESPONSE_SCHEMA=./prompts/report-response.schema.json
```

## اختبار الجودة

لتشغيل الاختبارات البرمجية:

```bash
python -m unittest discover -s tests -v
```

ومن لوحة التحكم يمكن تشغيل:

```text
تشغيل 10 اختبارات جودة محلية
```

نتائج الاختبارات تحفظ محليًا داخل `qa-results/` ولا ترفع إلى GitHub.

## الخادم والعامل المحلي

الخدمة المرجعية للطابور موجودة في `report_worker/server.py` وتستخدم SQLite.
تصلح للتجربة والنسخة الأولى الصغيرة:

```bash
uvicorn report_worker.server:app --host 0.0.0.0 --port 8080
```

إذا أردت أن ترسل واجهة Netlify الطلبات إلى هذا الخادم، اضبط أصل الواجهة في:

```bash
CONTROL_PLANE_ALLOWED_ORIGINS=https://your-site.netlify.app
```

العامل المحلي يسحب المهام من الخادم بدل أن يتصل الخادم بجهازك مباشرة:

```bash
python -m report_worker.cli run
```

عقد الاتصال موثق في:

```text
contracts/control-plane-openapi.yaml
```

## خطة المنتج

الاختيار التجاري الموصى به:

1. إبقاء Telegram كقناة سريعة للعملاء.
2. بناء موقع PWA بهيئة تطبيق لإدارة الحسابات والاشتراكات والهوية والملفات.
3. استخدام عامل محلي لتنفيذ النماذج وبناء PDF على جهازك.
4. تغليف الواجهة كتطبيق iOS/Android لاحقًا عند ثبوت الحاجة.

واجهة PWA الأولية موجودة في:

```text
webapp/
```

وإعداد Netlify موجود في:

```text
netlify.toml
```

إعداد النشر الحالي: `publish = "webapp"` بدون أمر بناء. هذا يعني أن Netlify
ينشر الواجهة فقط، بينما يبقى API والطابور وبناء PDF خارج Netlify.

لا تضع مفاتيح Telegram أو Gemini في Netlify لهذه الواجهة. المفاتيح تبقى في
لوحة التحكم المحلية أو في الخادم الخلفي عند نقله للإنتاج.

واجهة Netlify مخصصة للعملاء فقط؛ لا تعرض اختيار النموذج أو مفاتيح Gemini أو
Telegram أو إعدادات العامل المحلي. هذه الخيارات تبقى في لوحة التحكم الخاصة.

تفاصيل إضافية في:

```text
docs/DEPLOYMENT.md
```

## الأمان

لا ترفع هذه الملفات أو المجلدات:

- `.env`
- `data/settings.json`
- `data/assets/`
- ملفات PDF الناتجة
- قواعد SQLite المحلية
- `qa-results/`

قواعد `.gitignore` و`.dockerignore` مضبوطة لحماية هذه الملفات.

## ملاحظات LaTeX

يستخدم النظام LuaLaTeX لإخراج التقارير العربية. يفضل توفر خط `Tajawal` أو
بديل عربي مناسب على الجهاز. في حال فشل PDF راجع رسالة البوت أو سجل البناء
المحلي، فالنظام يحاول اختصار أخطاء LaTeX إلى رسالة مفهومة.
