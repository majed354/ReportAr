# خطة النشر التجاري

هذا المشروع مصمم ليعمل بنمطين متكاملين:

1. واجهة محلية سريعة عبر لوحة التحكم وبوت Telegram.
2. خادم تجاري يستقبل الطلبات ويدير الحسابات، بينما يسحب العامل المحلي المهام
   وينفذ النموذج وبناء PDF على جهازك.

## المكونات

- `report_worker/dashboard.py`: لوحة تحكم محلية لحفظ المفاتيح وتشغيل البوت.
- `report_worker/telegram_bot.py`: واجهة Telegram للمستخدمين.
- `report_worker/server.py`: خادم طابور مرجعي للتجربة والنسخة الأولى الصغيرة.
- `report_worker/worker.py`: عامل محلي يسحب المهام من الخادم.
- `report_worker/renderer.py`: محرك LuaLaTeX لإخراج PDF عربي منسق.
- `prompts/`: تعليمات النموذج ومخطط JSON الرسمي.

## ما لا يرفع إلى GitHub

- `.env`
- `data/settings.json`
- `data/assets/`
- قواعد SQLite المحلية
- ملفات PDF وLaTeX الناتجة
- نتائج اختبارات الجودة داخل `qa-results/`

## الإطلاق الموصى به

ابدأ بـ PWA وموقع تجاري يستخدم نفس الخادم، مع إبقاء Telegram كقناة سريعة.
بعد ثبات الاشتراكات وتجربة المستخدم يمكن تغليف الواجهة كتطبيق iOS/Android عبر
Capacitor دون إعادة بناء المحرك.

## المتطلبات الإنتاجية قبل البيع

- تسجيل دخول وحسابات مستخدمين.
- نظام رصيد أو اشتراك.
- مزود دفع سعودي.
- حدود استخدام لكل مستخدم.
- حفظ تاريخ الطلبات والملفات.
- صفحة هوية بصرية لكل مستخدم.
- مراقبة أخطاء PDF والنموذج.
- نسخ احتياطي لقاعدة البيانات.
