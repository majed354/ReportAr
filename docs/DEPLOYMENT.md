# خطة النشر التجاري

هذا المشروع مصمم ليعمل بنمطين متكاملين:

1. واجهة محلية سريعة عبر لوحة التحكم وبوت Telegram.
2. خادم تجاري يستقبل الطلبات ويدير الحسابات، بينما يسحب العامل المحلي المهام
   وينفذ النموذج وبناء PDF على جهازك.

## المكونات

- `report_worker/dashboard.py`: لوحة تحكم محلية لحفظ المفاتيح وتشغيل البوت.
- `report_worker/telegram_bot.py`: واجهة Telegram للمستخدمين.
- `report_worker/server.py`: خادم طابور مرجعي للتجربة والنسخة الأولى الصغيرة.
- `report_worker/worker.py`: عامل محلي يسحب المهام من الخادم.
- `report_worker/renderer.py`: محرك LuaLaTeX لإخراج PDF عربي منسق.
- `prompts/`: تعليمات النموذج ومخطط JSON الرسمي.

## ما لا يرفع إلى GitHub

- `.env`
- `data/settings.json`
- `data/assets/`
- قواعد SQLite المحلية
- ملفات PDF وLaTeX الناتجة
- نتائج اختبارات الجودة داخل `qa-results/`

## الإطلاق الموصى به

ابدأ بـ PWA وموقع تجاري يستخدم نفس الخادم، مع إبقاء Telegram كقناة سريعة.
بعد ثبات الاشتراكات وتجربة المستخدم يمكن تغليف الواجهة كتطبيق iOS/Android عبر
Capacitor دون إعادة بناء المحرك.

## Netlify

يرتبط GitHub مع Netlify لنشر الواجهة فقط:

- مجلد النشر: `webapp`
- ملف التهيئة: `netlify.toml`
- أمر البناء: لا يوجد في النسخة الثابتة الحالية

الواجهة لا تحتوي على مفاتيح Telegram أو Gemini. الربط مع الخادم يتم عبر إعداد
`API URL` من داخل الواجهة أثناء التجربة، ثم يستبدل لاحقًا بتسجيل دخول حقيقي
ونطاق API ثابت.

واجهة العملاء لا تعرض اختيار النموذج أو إعدادات المزودات. اختيار المحلي أو
Gemini والتحويل الاحتياطي يبقى في لوحة الإدارة فقط.

قبل أن يرسل زر الطلبات مهامًا حقيقية، يجب تشغيل API على الخادم وضبط:

```bash
CONTROL_PLANE_ALLOWED_ORIGINS=https://your-site.netlify.app
```

بدون API سيحفظ الموقع الطلب كمعاينة داخل المتصفح فقط.

لا تضف مفاتيح Telegram أو Gemini إلى Netlify في هذه المرحلة. Netlify يستضيف
واجهة عامة فقط. المفاتيح توضع في:

- لوحة التحكم المحلية عند تشغيل البوت محليًا.
- متغيرات بيئة الخادم الخلفي إذا نقلنا استدعاءات Gemini أو Telegram إليه لاحقًا.
- متغيرات Netlify فقط إذا أنشأنا Netlify Functions خاصة بالخادم، وهذا ليس
  التصميم الحالي.

## المتطلبات الإنتاجية قبل البيع

- تسجيل دخول وحسابات مستخدمين.
- نظام رصيد أو اشتراك.
- مزود دفع سعودي.
- حدود استخدام لكل مستخدم.
- حفظ تاريخ الطلبات والملفات.
- صفحة هوية بصرية لكل مستخدم.
- مراقبة أخطاء PDF والنموذج.
- نسخ احتياطي لقاعدة البيانات.
